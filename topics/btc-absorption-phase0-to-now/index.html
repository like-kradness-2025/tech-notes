何を見つけたかったか

いちばん最初の狙いは、BTCが売られたのに、あまり下がらない場面を見つけることだった。 これは、売りを誰かが受け止めている状態で、いわゆる吸収っぽい動き。

噛み砕くと、「売りが強そうなのに、価格が踏ん張る瞬間」 を拾いたかった。 そこを取れれば、反発を先回りしやすい。

技術的には、1秒粒度の features_1s を軸にして、 吸収っぽい局面をスコア化し、30分後・1時間後に勝っているかを検証する流れで進めた。

Phase 0: まず土台を作る

Phase 0 は、ざっくり言うと「怪しい場所を拾えるか」 を見る段階だった。 まだ荒いけれど、板・価格・出来高の基本形を使って、吸収の原型を作った。

たとえると、最初は地図の上に付箋を貼るようなもの。 どこが怪しいかは分かるが、まだ精密ではない。

技術面では、板圧力・出来高偏り・価格反応をベースに特徴量を組み、 Phase 0 バックテストで「反発点の吸収」が完全に外れではないことを確認した。 ただし、安定したシグナルというより、芽がある という段階だった。

Phase 1前半: 条件を増やしすぎた

次に、精度を上げようとして条件をどんどん足した。 ところが、条件が厳しすぎて、全部そろう場面が少なすぎる という問題にぶつかった。

噛み砕くと、 「見つけたいものが細かすぎて、ほとんど見えなくなった」 状態。

技術的には、F5 / F6 / F7 / OI view を積み上げたが、 AND条件が強すぎて通過率が落ち、特に F7 は最初ほぼ死んでいた。 ここで、単純に条件を増やせばよいわけではなく、 生きている特徴量と死んでいる特徴量を分ける必要 が見えてきた。

再設計: 4-viewに整理した

そこで、バラバラに盛るのをやめて、役割ごとに4つの視点へ整理した。 これで、シグナルが何を見ているのかがはっきりした。

4つの視点は次の通り。

  • Orderflow Absorption — 売りを受け止めているか
  • OI Confirmation — 建玉の動きが本物か
  • Book Depth Support/Resistance — 板が支えているか
  • Context / Location — VWAPやTPOでどんな場所か

噛み砕くと、「値動きだけではなく、場所と状況も見る」 形にした。 これで、単純な板シグナルから、文脈込みの吸収判定に進めた。

技術的には、ノイズ特徴量を削り、TPO / VWAP を位置情報として追加。 役割ごとに特徴量を分けたことで、何が効いているか追いやすくなった。

ラベルを見直して、勝てる塊を探した

次の課題は、勝ち負けの判定そのもの だった。 固定bpsの判定だと相場環境で歪むので、ATR相対ラベルに切り替えた。

噛み砕くと、 「絶対値ではなく、そのときの相場に対してどれくらい動いたか」 で見るようにした、ということ。

技術的には、ATR相対化でラベルを再定義し、 +1 ラベルが大きく増えた。 さらに forward stats、walk-forward の FN、recall 分母などのバグも修正して、 検証がちゃんと回るように整えた。

その結果、上位パーセンタイルでまとまって勝てる塊が見えた。 特に P98〜P99 あたりで、件数は少ないが勝率はかなり良い。

負ける35%をどう減らすか

6割勝てるなら十分強いように見えるが、実戦では残りの負けが気になる。 なので次は、当たりを増やす より、負けを避ける 方針になった。

噛み砕くと、 「怪しい場面は最初から取らない」 という考え方。

負けやすい特徴として目立ったのは次の通り。

  • VWAPから遠い
  • レンジの端すぎる
  • 東京時間の昼など、だましが増えやすい
  • 入っても15分で反応しない

ここから、15分 early-exit と 時間帯フィルタ が有力になった。 つまり、入ったあとも見張る のではなく、 最初から悪い場面を減らす 方向に進んだ。

今の結論

いまの整理はシンプルで、吸収シグナルは 「売られても落ちない場所を拾う道具」 になった。

噛み砕くと、 場所・時間・反応 の3点が大事。 ただ強そうに見えるだけではだめで、 ちょうどいい場所で、ちょうどいい時間に、ちゃんと反応するか を見ないといけない。

技術的には、Phase 0 から始まって、 特徴量の再設計、ラベルの再定義、バックテストの修正を経て、 今は features_1s ベースでかなり筋の良いシグナルが見えている。 次の焦点は、負ける30〜35%をどう削るか だ。
