吸収とは何か

BTCのオーダーフロートレードにおいて、「価格が下がっているのに売り圧力が吸収されている」瞬間を捉えたい。 大量の売り成行が入れば普通は価格が下がる。でも下がらない——誰かが指値で全部買っている。これが「吸収（Absorption）」。 吸収が終わると溜まったエネルギーが解放されて反発（または崩壊）する。

このノートは、その「吸収＝反発」パターンを特徴量として設計し、実際のデータ（features_1s, BingX perpetual, 222,947行）で検証した記録。

ラベルの定義（グランドトゥルース）

「吸収のあとに反発した」を事後的に4つのラベルで定義する。

+1（強気反発吸収）: 売り成行急増 + CRR正常 + 壁維持 + 120秒後に0.03%以上上昇
-1（弱気反転吸収）: 買い成行急増 + CRR正常 + 壁維持 + 120秒後に0.03%以上下落
 0（該当なし）    : 上記以外
-2（フェイクアウト）: 成行急増 + キーレベル突破 + 回復なし

キーレベルは「直近15分のスイング高値/安値」。超えて戻らないとフェイクアウト判定。

4つの指標（Book Depth）

まずは板のデータ（features_1s, 1秒分解能）だけで検証した。これがPhase 0。

指標なにを測るか優しい言い方
CRR成行消費 ÷ 指値補充「攻撃に対して板が補充できてるか」
WPS板の厚みが30秒安定してるか「壁が持ちこたえてるか」
RVZ補充速度の異常スコア(Z値)「いつもより明らかに補充が早いか」
TMR成行消費 ÷ 指値取消「低い＝板防御中／高い＝崩壊」

どう判定するか — 重み付きスコア

4つの指標をそのままANDで組むと候補がゼロになる。そこで重み付きスコアを使う。

CRR_bid 0.01〜0.1 → +0.35点（←これがないとダメ）
WPS_bid 0.7超     → +0.30点
TMR_bid 0.01未満 → +0.20点（←低いほうが良い！）
RVZ_bid 1.0超     → +0.15点
────────────────────────
合計 0.60以上 で「吸収シグナル」

さらに sell_notional > P90(120s) ← 必須条件
なぜスコア方式か
AND条件だと「CRRはOKだけどWPSが微妙」が全部落ちる。スコア方式なら「CRR+WPS+TMRが強い」場合にRVZが弱くても通る。実データの分布に合わせた現実的な設計。

ハマったポイント3つ

① TMRの方向を逆にしていた（致命的）

TMRは「低い=良い（板が防御できてる）」。なのに最初は「TMR > 0.5」でエントリー条件に。 0.5はP99の13倍、現実的に到達不能。意味も逆。ここで半日溶かした。

→ 修正: TMR_bid > 0.5 → TMR_bid < 0.01

② RVZの閾値が高すぎた

RVZ > 3.0と設定していたが、P95相当。CRR+WPSの847候補にこれでフィルタすると2件になった。

→ 修正: RVZ > 3.0 → RVZ > 1.0（P90相当）

③ フォワードリターンの閾値が高すぎた

120秒で+0.1%を「反発」の基準にしていたが、レンジ相場ではmax forwardが0.087%で達しない。

→ 修正: 60s/0.1% → 120s/0.03%

トレンド相場の方がシグナルが安定する。レンジだと15分までは機能するが1時間後には戻される。 逆に言えば「レンジでは短期逆張り、トレンドでは順張りホールド」と住み分け可能。

フェイクアウトはバッチリ検出

CRR崩壊（消費はあるのに補充ゼロ、CRR=999）の検出率: 97.9%。
売り成行の急増 + 補充なし = 板が抜けるパターンをほぼ完璧に押さえられている。

まとめ

  • ✅ 当たると強い: 的中時の15分-1時間勝率67-77%
  • ✅ フェイクは弾く: CRR崩壊検出97.9%
  • ⚠️ 数は少ない: 全データの0.03%しか+1イベントがない
  • ⚠️ 3回に2回はハズレ: Precision 14-22%（改善の余地あり）
  • 💡 トレンド相場向き: レンジだと長期リターンが減衰

この後どう進むか

  3. Phase 3: 実取引で使えるか、コスト込みでwalk-forward検証
