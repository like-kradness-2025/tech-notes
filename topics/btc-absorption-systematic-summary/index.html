まず全体像

このプロジェクトは、BTCが売られた局面で「売りが強いのに価格が崩れない」場面を見つけるために始まった。 いわゆる吸収（absorption）を、勘ではなくデータで拾うのが目的だった。

いまの整理では、シグナルは単なる「板の異常検知」ではない。 データ分析 → 位置的文脈 → 時間的優位性 の3層で見ないと、実戦では弱い。

ざっくり言うと、 何が起きているか をデータで見つけ、 どこで起きているか を位置で判断し、 いつ起きているか を時間で絞る。

  ↓

Phase 0 から今までの流れ

Phase 0 では、まず板データの基本形から始めた。 ここで重要だったのは、吸収が本当に存在するのか を確認すること。 まだ粗いが、板・出来高・価格反応の組み合わせで、反発候補が完全な幻想ではないと分かった。

その後の Phase 1 前半では、精度を上げようとして条件を増やしたが、 条件が厳しすぎて候補がほぼ消えた。 ここで「良い条件を足せば良いシグナルになる」という考えが崩れた。

そこで、視点を役割別に分ける形に再設計した。 最終的には、候補生成＋スコアリング の形に寄せ、 ラベルも固定bpsではなく ATR相対 に再定義した。

つまり流れとしては、 「吸収はある」→「でも条件を盛りすぎると死ぬ」→「役割分担と文脈付けで再構築」 という進化だった。

1. データ分析層

ここは、“吸収っぽいものをデータから見つける層”。 何より先に必要なのは、シグナルが本当にあるかどうかを確認すること。

Phase 0 の役割

Phase 0 では、まず板の深さや消費・補充のバランスを見た。 代表的な考え方は次の4つ。

指標見るもの意味
CRR成行消費 ÷ 指値補充板が攻撃に耐えているか
WPS板の厚みの安定性壁が維持されているか
RVZ補充速度の異常度いつもより早く補充されているか
TMR成行消費 ÷ 指値取消板が防御しているか、崩れているか

噛み砕くと、売りが来ても板が持ちこたえているか を見ていた。 これが吸収の原型になる。

Phase 1 で何が起きたか

次に、CVD や OI を加えて、より多面的に見ようとした。 しかし、条件を足しすぎたことで、正しいけれど疎な条件同士の AND が候補を消してしまった。

ここで重要だったのは、全部を必須にする設計は危険 だと分かったこと。 特に、通過率が極端に低い条件をゲートに置くと、何も残らない。

再設計の方針

そこで、特徴量は役割ごとに整理し、最終判定は「AND」ではなく 候補生成 → スコアリング に変えた。 ここでの役割分担は、概ね次の4つ。

  • Orderflow Absorption — 売りを受け止めているか
  • OI Confirmation — 建玉の変化が本物か
  • Book Depth Support/Resistance — 板が支えているか
  • Context / Location — どんな場所で起きているか

この整理によって、単純な板の異常ではなく、 “吸収らしさ” を多視点で合成する設計 になった。

ラベルの見直しが効いた

固定bpsのラベルは、相場環境によって歪みやすかった。 そこで ATR相対ラベル に切り替えたことで、 「その場でどれくらい動いたか」を相場に対して相対的に見られるようになった。

この変更で、正例の件数は大きく増え、検証の意味が出た。 つまり、ラベルが定常化すると、特徴量の良し悪しが見えるようになる。

データ分析層の結論

この層の役割は、シグナル候補を見つけること。 ただし、候補を増やすだけでは足りない。 候補の中から、文脈と時間で落とし込む必要がある。

2. 位置的文脈層

次の層は、“そのシグナルがどこで出たのか” を見る層。 同じ吸収の形でも、出た場所が違えば意味が変わる。

ここで使うのが、VWAP や TPO、POC、直近レンジ、価格乖離といった位置情報。 吸収は、価格が市場の中心に近いか、端に寄りすぎているか で強さが変わる。

なぜ場所が重要か

価格がみんなに見られている場所の近くなら、参加者が多い。 逆に、平均から遠すぎる場所では、支えが効いても無理筋になりやすい。

噛み砕くと、 “同じ吸収でも、駅前で起きるのと人気のない道で起きるのは違う” ということ。

代表的な位置指標

指標何を見るか実務上の意味
VWAP取引の平均価格市場参加者の中心
TPO / POC時間帯別の受け入れ位置よく売買された価格帯
直近レンジ最近の高値安値端に寄りすぎていないか
価格乖離平均からのズレ無理な場所かどうか

位置的文脈の結論

位置的文脈は、「この吸収は、意味のある場所で起きたか」 を判断するための層。 ここが悪いと、データ上は吸収っぽく見えても、実戦ではだましになりやすい。

3. 時間的優位性層

3つ目の層は、“いつ起きたか” を見る層。 これは単なる曜日・時間帯の話ではなく、反応速度と持続性 も含む。

時間帯による差

同じ形でも、時間帯で成績が変わる。 とくに東京時間の昼はだましが増えやすく、逆に夕方以降は素直に伸びる場面がある。 つまり、市場は24時間同じ性格ではない。

15分 early-exit の考え方

吸収が本物なら、入ったあとに比較的早く反応する。 15分たっても含み損のままなら、思ったほど買いが強くない可能性が高い。

ここでの重要点は、時間はエントリー条件であると同時に、出口条件でもある こと。 つまり「いつ入るか」だけでなく「いつ見切るか」が、時間的優位性の一部になる。

walk-forward の意味

時間的優位性は、単発のバックテストではなく、walk-forward で確認する必要がある。 その場では強く見えても、別期間で崩れるなら、本当に時間の優位性があるとは言えない。

時間的優位性の結論

時間層は、「入る時間」 と 「反応の時間」 を同時に扱う。 これで、勝てる局面を増やすというより、負ける局面を減らす 方向に効く。

今の実務フロー

ここまでをまとめると、現在の実務フローはかなり明確になっている。 先に候補を取り、次に場所、最後に時間で落とす。

工程見ること落とすもの
1. 候補生成吸収らしい板・出来高・OIの異常何も起きていないもの
2. 位置フィルタVWAP / TPO / 価格乖離 / レンジ遠すぎる・端すぎる場所
3. 時間フィルタ時間帯 / 反応速度 / walk-forwardだましが増える時間帯
4. 退出制御15分 early-exit反応しない負けシナリオ

実務上は、P98〜P99級の上位スコアだけ残す 方針が自然だった。 頻度を上げるより、精度を上げるほうが理にかなっている。

ここまでの学び

  • データ分析だけでは足りない — 形が良くても場所が悪ければ弱い
  • 位置だけでも足りない — どこで起きても、反応が遅ければ負ける
  • 時間だけでも足りない — 良い時間でも、候補が雑なら意味がない
  • 疎な条件をANDしすぎると死ぬ — 候補を消さず、スコアで絞るほうが良い
  • ラベル定義は特徴量と同じくらい重要 — ATR相対化で見え方が変わった

次に残る課題

いまのシグナルは、かなり筋が良いところまで来ている。 ただし、まだ最後の詰めがある。

  1. 負ける30〜35%を、位置と時間でさらに削る
  2. long / short を分けて、方向別に最適化する
  3. より長い期間で walk-forward を積み上げる
  4. 必要な場合だけ生データへ降りる（順序情報・大口/小口判定など）

つまり、今は「シグナルを作る段階」から、“どのシグナルだけ残すか” を詰める段階 に入っている。
